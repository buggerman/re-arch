The Re-Arch Procedure

Transform minimal Arch Linux into an optimized desktop powerhouse

🚀 Lite Approach (Recommended)

Revolutionary two-step process: 80% work in archinstall, 20% final configuration

💡 Tip: You can customize username, passwords, and other settings when archinstall opens. Default credentials (user/rearch) are provided as fallback to prevent lockouts.

Step 1: Enhanced archinstall

archinstall --config-url https://raw.githubusercontent.com/buggerman/re-arch/main/archinstall-config.json --creds-url https://raw.githubusercontent.com/buggerman/re-arch/main/archinstall-credentials.json

Step 2: Lite configuration

arch-chroot /mnt && curl -fsSL https://raw.githubusercontent.com/buggerman/re-arch/main/re-arch-lite.sh | bash

🎯 Why Choose Lite?

  • 🚀 95% fewer failure points
  • ⚡ 2-3 minutes vs 15-30 minutes
  • 🛡️ Maximum reliability via archinstall
  • ✨ Same optimized desktop result

⚠️ Important Note

The lite script performs final system configuration. Run Step 2 only after completing the archinstall process and chrooting into your new system.

Complete Installation Workflow

From bare metal to production desktop in minutes

1

Install Arch Linux

Use archinstall with our provided configuration for automated base system setup with Btrfs.

2

Chroot Environment

Boot from installation media, mount your system, and chroot into the installed environment.

3

Run Re-Arch

Execute our one-command installer. The script handles everything else automatically.

4

Reboot & Enjoy

Reboot into your optimized system with KDE Plasma, snapshots, and performance tuning.
