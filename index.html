The Re-Arch Procedure

Complete Arch Linux desktop installer with curated optimizations

🚀 Two-Step Installation (Recommended)

🕐 Total time: 25-35 minutes | 📶 Internet required throughout

🚨 DESTRUCTIVE INSTALLATION WARNING

This installer assumes it controls the entire disk. It WILL erase everything on the target disk. Back up important data BEFORE starting.

📋 BEFORE YOU START:

🔲 Boot from Arch Linux ISO (download from archlinux.org)
🔲 Connect to internet: For WiFi: iwctl station wlan0 connect "YourNetwork"
🔲 Test connection: ping google.com

⚠️ Important: What NOT to Change

❌ Packages - Don't modify the package list unless you know what you're doing
❌ Disk formatting - Only change if you need dual boot or custom partitioning
❌ Filesystem - Keep Btrfs for snapshot functionality to work
❌ Bootloader - Keep GRUB for snapshot boot integration

Safe to change: Username, passwords, timezone, locale, hostname

🔐 Default Credentials: Username: user | Password: rearch | Root: rearch

Change these during archinstall setup or after first login for security

🚀 STEP 1: Automated Installation (20-30 minutes)

Run this command as root:

archinstall --config-url https://re-arch.xyz/config.json --creds-url https://re-arch.xyz/creds.json

✅ Success indicators: "Installation completed successfully" + prompt asking about chroot

⚙️ STEP 2: System Optimization (2-3 minutes)

🤔 Which method to use?

Method A (RECOMMENDED): When archinstall asks about chroot, select "Yes" → You'll see (chroot) prompt → Run script

Method B: If you selected "No" → Run arch-chroot /mnt → See (chroot) prompt → Run script

curl -fsSL https://re-arch.xyz/re-arch-lite.sh | bash

✅ Success indicators: "Re-Arch lite configuration completed successfully" + no error messages

🔄 Alternative URLs (if domain is not working)

Step 1:

archinstall --config-url https://raw.githubusercontent.com/buggerman/re-arch/main/config.json --creds-url https://raw.githubusercontent.com/buggerman/re-arch/main/creds.json

Step 2:

curl -fsSL https://raw.githubusercontent.com/buggerman/re-arch/main/re-arch-lite.sh | bash

🎉 After Installation Complete

1. Exit chroot: Type exit

2. Unmount: Run umount -R /mnt (if you manually chrooted)

3. Reboot: Remove USB/CD and run reboot

📋 System Requirements & Troubleshooting

💻 Hardware Requirements

  • CPU: x86_64 processor (Intel/AMD 64-bit)
  • RAM: 4GB minimum, 8GB recommended
  • Storage: 20GB minimum, 50GB recommended
  • Boot: UEFI preferred, Legacy BIOS supported

✅ What's Supported

  • ✅ Fresh installations on dedicated machines
  • ✅ Virtual machines (VirtualBox, VMware, etc.)
  • ⚠️ Dual-boot systems (advanced users only)
  • ❌ Systems with existing data to preserve

🚨 Common Issues

  • No internet: Connect WiFi before starting
  • Disk space errors: Ensure 20GB+ available
  • Boot issues: Check UEFI/BIOS settings
  • Package failures: Retry with fresh ISO

🆘 Getting Help

  • GitHub Issues: Report bugs
  • Arch Wiki: General guidance
  • System recovery: Boot from snapshots in GRUB
  • Check logs: journalctl -b

⚗️ Experimental: Legacy Conversion Method

⚠️ WARNING: NOT RECOMMENDED

This method is failure-prone and should only be used by advanced users for converting existing minimal Arch installations.

Prerequisites for Conversion:

  • Existing minimal Arch Linux with Btrfs root
  • Non-root user with sudo privileges
  • Boot from Arch installation media
curl -fsSL https://re-arch.xyz/re-arch.sh | bash

Known Issues:

  • Higher failure rates due to package conflicts
  • Complex dependency resolution required
  • Manual troubleshooting often needed
